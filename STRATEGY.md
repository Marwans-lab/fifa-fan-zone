### 2.2 Key Technical Components

#### A. Match Scheduler Service
**Purpose**: Orchestrate quiz delivery based on live match events

**Responsibilities**:
- Poll FIFA API every 30 seconds for match state
- Detect: Match Start, Halftime, Full Time, Goals, Cards
- Trigger quiz drops at key moments
- Handle timezone conversion for global users

**Tech Stack**: Node.js, Bull Queue, Redis, Cron

#### B. Quiz Distribution Engine
**Purpose**: Deliver the right quiz to the right user at the right time

**Logic**:
```javascript
if (match.team1 === user.favoriteTeam || match.team2 === user.favoriteTeam) {
  // Send team-specific quiz
  sendPushNotification(user, {
    title: `${user.favoriteTeam} is playing NOW! 🔥`,
    body: "Answer trivia and earn points!",
    deepLink: `/quiz/match-${match.id}`
  });
}
```

**Features**:
- Team-based filtering
- Geographic targeting (fans in specific countries)
- Frequency capping (max 6 notifications per day)
- Delivery optimization (avoid spamming)

#### C. Leaderboard Service (High Performance)
**Purpose**: Real-time rankings with millisecond updates

**Architecture**: Redis Sorted Sets
```javascript
// User submits quiz score
ZADD leaderboard:global [score] [userId]
ZADD leaderboard:team:dz [score] [userId]

// Get top 100
ZREVRANGE leaderboard:global 0 99 WITHSCORES

// Get user rank
ZREVRANK leaderboard:global [userId]
```

**Scaling Strategy**:
- Redis Cluster for horizontal scaling
- Read replicas for leaderboard queries
- Daily snapshots to Firestore for persistence

#### D. Reward Engine
**Purpose**: Award points, badges, Avios based on performance

**Reward Tiers**:
```
Match Completion:       10-50 points (score-based)
Perfect Score:          +20 bonus points
Speed Bonus:            +10 points (< 30 seconds)
Halftime Quiz:          +5 points (participation)
Streak (3 matches):     +50 points
Round Milestone:        +100 points (Round of 16, QF, SF)
Grand Prize Entry:      Auto at 500 points
```

**Avios Distribution**:
- Weekly: Top 10 get Avios (1000/500/250)
- Round Completion: Top 3 per round get bonus Avios
- Tournament End: Grand prize winner

### 3.2 Personalization Engine

**Data Collected** (with consent):
- Selected team (onboarding)
- Match notification preferences (all matches vs. my team only)
- Quiz performance (accuracy, speed, streaks)
- Engagement patterns (time of day, day of week, quiz types preferred)
- Social graph (friends who also use app)

**Personalization Layers**:

**L1: Team-Based**
- Content filtered by selected team ("Argentina" user sees Messi trivia, not Ronaldo)
- Notifications only for selected team matches (reduces spam)
- Leaderboard defaults to "Argentina fans" view (tribal competition)

**L2: Performance-Based**
- Adaptive difficulty (80%+ accuracy → harder questions, <40% → easier questions)
- Personalized challenges ("You're strong on player trivia, weak on stats — try this quiz")

**L3: Behavioral**
- Send notifications at user's preferred time (if they always play at 8pm, schedule then)
- Suggest quiz types they engage with most (predictions vs. history vs. stats)

**L4: Social**
- "Your friend John just beat your high score on Brazil vs. Argentina"
- "3 of your friends are taking the Final quiz now — join them!"

**Privacy Principles**:
- All data stored in user's Firestore document (not shared without consent)
- Opt-out available ("Disable personalization" in settings)
- GDPR/CCPA compliant (right to deletion)

### 3.4 Retention Tactics (Anti-Churn)

**Churn Signals** (user about to drop off):
- No activity for 3 days during tournament
- Notification open rate drops below 20%
- Quiz completion rate <30% (starts but doesn't finish)
- Leaderboard rank plateaus (not improving)

**Re-Engagement Campaigns**:

**Scenario 1: 3-Day Inactive**
- Push: "We miss you! Your team plays tomorrow — predict the winner & earn 100 bonus points"
- Email: "You're still in the top 5000 — don't lose your grand prize chance"

**Scenario 2: Low Quiz Completion**
- In-app: "Quizzes too hard? Try our 'Fun Facts' quiz (easy mode)"
- Adaptive difficulty: Auto-switch to easier questions

**Scenario 3: Leaderboard Plateau**
- Push: "Stuck at #4,523? Here's a challenge quiz worth 2x points"
- Social: "Your friend Sarah climbed 200 ranks this week — challenge her!"

**Scenario 4: Notification Fatigue**
- Smart throttling: If user ignores 5 consecutive notifications, reduce frequency to 1/day
- Preference prompt: "Too many notifications? Customize your alert settings"

**Success Metric**: Reduce churn rate by 20% vs. baseline (no re-engagement)

## 6. Security & Compliance

### 6.1 Security Measures

**A. Anti-Fraud / Anti-Cheating**:

**Problem**: Users may try to game the system (bots, multiple accounts, answer sharing)

**Solutions**:
1. **Rate Limiting**:
   - Max 1 quiz per user per match
   - Max 10 quiz attempts per day
   - IP-based throttling (max 100 req/min per IP)

2. **Device Fingerprinting**:
   - Track device ID (iOS/Android)
   - Flag multiple accounts from same device
   - Require re-authentication after 3 accounts on one device

3. **Answer Timing Analysis**:
   - Flag users who answer all questions in <5 seconds
   - Flag users with 100% accuracy across 10+ quizzes (suspicious)
   - Manual review queue for flagged accounts

4. **CAPTCHA** (Last Resort):
   - Show reCAPTCHA v3 if user is flagged
   - Only for high-value actions (claiming rewards)

**B. Data Privacy (GDPR / CCPA Compliance)**:

**User Data Collected**:
- Email (if signed in)
- Device ID
- Location (city-level for notifications)
- Quiz answers & scores
- Notification preferences

**Privacy Controls**:
- **Consent**: Explicit opt-in for notifications and data processing
- **Data Export**: User can download all their data (JSON)
- **Right to Deletion**: User can delete account & all data
- **Anonymization**: Quiz answers stored with anonymized user ID

**Legal Requirements**:
- Privacy Policy (updated)

### 10.2 Architectural Decisions That Enable Scalability

**Decision 1: Universal Event Schema** (not "FIFA Match Schema")

```typescript
// ❌ Wrong: FIFA-specific (doesn't scale)
interface FIFAMatch {
  homeTeam: string;
  awayTeam: string;
  kickoffTime: Date;
  venue: string;
}

// ✅ Right: Universal (works for soccer, F1, tennis, concerts)
interface UniversalEvent {
  id: string;
  partnership: "fifa" | "psg" | "formula_e" | "tennis" | "concert";
  category: "sports" | "entertainment" | "lifestyle";
  name: string;                    // "Argentina vs. France" or "Coldplay Paris"
  date: Date;
  venue?: string;
  participants: string[];          // ["Argentina", "France"] or ["Coldplay"]
  quizTemplates: QuizTemplate[];   // Same structure for all event types
  rewardMultiplier: number;
}
```

**Why This Matters**: 
- PSG match = change `partnership: "psg"`, everything else identical
- Formula E race = change `participants: ["DS Penske", "Jaguar"]`, same quiz engine
- Concert = change `category: "entertainment"`, same UI components

**FIFA Implementation**: We build UniversalEvent first, populate it with FIFA data → no refactor needed for Phase 2

---

**Decision 2: AI-Powered Content Generation** (not manual-only)

**FIFA Content Volume**: 2,080 questions (104 matches × 5 questions × 4 quizzes)

**Traditional Approach**: 
- Hire 5 content editors → $50K/year × 5 = $250K
- 3 months of manual writing
- Doesn't scale (PSG = 50 matches = 1,000 more questions = +2 editors)

**Scalable Approach**:
```python
# Universal quiz prompt that works for ANY event

def generate_quiz(event: UniversalEvent, quiz_type: str):
    prompt = f"""
    You are writing trivia for Qatar Airways' fan engagement platform.
    
    Event Type: {event.category}
    Event Name: {event.name}
    Participants: {", ".join(event.participants)}
    Quiz Type: {quiz_type}  # "pre", "live", "post"
    
    Generate 5 multiple-choice questions:
    1. Participant history/background
    2. Head-to-head or past performances
    3. Venue/location trivia
    4. Prediction question
    5. Qatar Airways brand tie-in (subtle)
    
    Format: JSON array with question, options (4), correctAnswer, explanation
    """
    
    return openai.complete(prompt)
```

**Result**: 
- FIFA: AI generates 80% of 2,080 questions → 2 editors review → $30K cost
- PSG: Same AI generates 1,000 questions → 0 additional editors → $0 marginal cost
- Formula E: Same AI generates 320 questions → $0 marginal cost

**FIFA Implementation**: Build AI generation in Phase 1, prove quality → reuse for all future events

**Decision 3: Plugin Handler Architecture** (not monolithic codebase)

**FIFA as a Plugin**:
```typescript
// fifa_handler.ts (150 lines)
class FIFAHandler implements PartnershipHandler {
  id = "fifa";
  
  async fetchEvents() {
    // Call FIFA API, transform to UniversalEvent[]
    const matches = await fifaAPI.getWorldCupMatches();
    return matches.map(m => ({ ...universalEventFormat }));
  }
  
  async generateQuiz(event: UniversalEvent) {
    // Use AI with FIFA-specific context
    return aiService.generateQuiz({ sport: "soccer", ...event });
  }
  
  formatDisplay(event: UniversalEvent) {
    // FIFA branding: colors, logos
    return { backgroundColor: "#1F4788", accentColor: "#FFD700" };
  }
}
```

**PSG as a Plugin** (same pattern, 2 days to build):
```typescript
// psg_handler.ts (120 lines)
class PSGHandler implements PartnershipHandler {
  id = "psg";
  
  async fetchEvents() {
    const matches = await uefaAPI.getTeamFixtures("PSG");
    return matches.map(m => ({ ...universalEventFormat }));
  }
  
  async generateQuiz(event: UniversalEvent) {
    return aiService.generateQuiz({ sport: "soccer", ...event });
  }
  
  formatDisplay(event: UniversalEvent) {
    return { backgroundColor: "#004170", accentColor: "#DA020E" };
  }
}
```

**Why This Matters**:
- Core platform (scheduler, quiz engine, leaderboard, rewards) = 90% of code
- Handlers (FIFA, PSG, Formula E) = 10% of code, 100 lines each
- Adding PSG = 2 days of work, not 3 months

**FIFA Implementation**: Build core platform + FIFA handler in Phase 1 → validate handler pattern works

**Decision 4: Stateless Quiz Engine** (not FIFA-hardcoded logic)

```typescript
// ❌ Wrong: FIFA-specific quiz logic
function calculateScore(answer: Answer) {
  if (isFIFAMatch && isCorrect) return 50;
  if (isFIFAFinal && isCorrect) return 100;
  // ... doesn't work for PSG or Formula E
}

// ✅ Right: Event-agnostic scoring
function calculateScore(answer: Answer, event: UniversalEvent) {
  const basePoints = 50;
  const multiplier = event.rewardMultiplier; // 1.0 for group stage, 2.0 for final
  const speedBonus = calculateSpeedBonus(answer.timeSpent);
  
  return (basePoints * multiplier) + speedBonus;
}
```

**Why This Matters**: 
- FIFA Final uses `rewardMultiplier: 2.0`
- PSG Champions League Final uses `rewardMultiplier: 2.0`
- Same code, different config

**FIFA Implementation**: Build scoring engine with multipliers in Phase 1 → no changes needed for Phase 2
