## Executive Summary

Transform the FIFA Fan Zone investment into a **single, reusable, plug-and-play platform** that activates ALL Qatar Airways sponsorships and Privilege Club Collection experiences through one unified digital engagement layer.

**Core Principle**: **Build Once, Deploy Everywhere**
- One codebase that adapts to ANY event type (soccer, F1, tennis, concerts, dining)
- Same core mechanics (quizzes, predictions, leaderboards, Avios rewards)
- Event-agnostic architecture with sport/experience-specific templates
- Zero custom development per event — configuration-driven activation

**Strategic Rationale**:
- Qatar Airways already pays **$100M+ annually** in sports/event sponsorships
- Most activations are physical-only (signage, hospitality, branding)
- **Digital engagement gap**: No unified mobile experience across partnerships
- **This platform**: Turn every sponsored event into a fan engagement opportunity with zero marginal engineering cost

**Business Case**:
- **Maximize existing assets**: No new sponsorship costs required
- **Scalability**: Add new event types with configuration, not code
- **Cross-sell opportunities**: Drive Privilege Club sign-ups, Avios redemptions
- **Data goldmine**: Understand fan preferences across all partnerships
- **Competitive differentiation**: Only airline with integrated event engagement
- **Cost efficiency**: $504K for FIFA → +10 sports/experiences for <$100K incremental

## 4. Technical Architecture (Plug-and-Play Universal System)

### 4.1 Core Principle: Configuration Over Code

**Problem with Traditional Approach**:
```
New event type → Write custom handlers → Test → Deploy → Maintain
= High cost per event, engineering bottleneck
```

**Plug-and-Play Approach**:
```
New event type → Add config file → Auto-generates UI/logic → Go live
= Zero engineering cost per event, instant activation
```

---

### 4.2 Universal Event Schema (Works for Everything)

```typescript
// Single event model that handles ALL Qatar Airways partnerships

interface UniversalEvent {
  // Metadata (same for all events)
  id: string;
  partnership: "fifa" | "psg" | "roma" | "bayern" | "formula_e" | "tennis" | "concert" | "dining";
  category: "sports" | "entertainment" | "lifestyle";
  
  // Event details (flexible)
  name: string;                    // "PSG vs. Real Madrid"
  date: Date;                      // When it happens
  venue?: string;                  // "Parc des Princes" (optional)
  participants: string[];          // ["PSG", "Real Madrid"] or ["Coldplay"] or ["Chef Gordon Ramsay"]
  
  // Engagement mechanics (universal)
  quizTemplates: QuizTemplate[];   // Pre-match, live, post-match, or pre-concert, post-experience
  predictionQuestions: Question[]; // Who will win? What will the score be?
  rewardMultiplier: number;        // 1.0 = standard, 2.0 = special events (finals)
  
  // Content (auto-generated or manual)
  coverImage: string;
  description: string;
  
  // Privilege Club integration
  privilegeClubExperienceId?: string;  // Link to redeemable experience
  aviosRequired?: number;              // If this event is redeemable
}

interface QuizTemplate {
  type: "pre" | "live" | "post";         // When it runs
  questions: Question[];                  // 5-10 questions
  autoGenerate: boolean;                  // Use AI or manual content?
  contentSource?: "api" | "manual";       // Where questions come from
}
```

**Key Insight**: Same structure works for:
- PSG vs. Bayern (sports_match)
- Formula E race (sports_match)
- Wimbledon final (sports_match)
- Coldplay concert (entertainment_experience)
- Michelin dining (lifestyle_experience)

Only `participants`, `quizTemplates`, and `contentSource` change — everything else is identical.

### 4.3 Plugin Architecture (Sport/Experience Handlers)

```typescript
// Each partnership gets a lightweight handler (not a custom codebase)

interface PartnershipHandler {
  id: string;                              // "psg", "formula_e", "concert"
  fetchEvents(): Promise<UniversalEvent[]>; // Get upcoming events
  generateQuiz(event: UniversalEvent): Promise<Question[]>; // Auto-create content
  formatDisplay(event: UniversalEvent): EventCard; // UI customization
}

// Example: PSG Handler (50 lines of code)
class PSGHandler implements PartnershipHandler {
  id = "psg";
  
  async fetchEvents() {
    // Call UEFA API for PSG fixtures
    const fixtures = await uefaAPI.getTeamFixtures("PSG");
    
    // Convert to UniversalEvent
    return fixtures.map(f => ({
      id: f.matchId,
      partnership: "psg",
      category: "sports",
      name: `${f.homeTeam} vs. ${f.awayTeam}`,
      date: f.kickoff,
      venue: f.stadium,
      participants: [f.homeTeam, f.awayTeam],
      quizTemplates: [
        { type: "pre", questions: [], autoGenerate: true, contentSource: "api" },
        { type: "live", questions: [], autoGenerate: false, contentSource: "manual" },
        { type: "post", questions: [], autoGenerate: true, contentSource: "api" }
      ],
      rewardMultiplier: f.competition === "Champions League" ? 2.0 : 1.0,
      coverImage: f.officialPoster,
      description: `Ligue 1 - Matchday ${f.matchday}`
    }));
  }
  
  async generateQuiz(event: UniversalEvent) {
    // Use AI to generate questions based on event.participants
    return aiService.generateQuiz({
      sport: "soccer",
      teams: event.participants,
      matchHistory: await this.getHeadToHead(event.participants)
    });
  }
  
  formatDisplay(event: UniversalEvent): EventCard {
    // PSG-specific colors, logo
    return {
      backgroundColor: "#004170", // PSG blue
      accentColor: "#DA020E",     // PSG red
      logo: "psg-logo.svg",
      teamFlags: event.participants.map(t => `flag-${t}.svg`)
    };
  }
}

// Example: Concert Handler (30 lines of code)
class ConcertHandler implements PartnershipHandler {
  id = "concert";
  
  async fetchEvents() {
    // Manual entry via CMS (no API for concerts)
    return cmsService.getConcerts({ status: "published" });
  }
  
  async generateQuiz(event: UniversalEvent) {
    // Artist trivia, venue history, setlist predictions
    return aiService.generateQuiz({
      category: "music",
      artist: event.participants[0],
      genre: event.metadata.genre
    });
  }
  
  formatDisplay(event: UniversalEvent): EventCard {
    return {
      backgroundColor: "#1DB954", // Spotify green (music theme)
      accentColor: "#FFFFFF",
      logo: "concert-icon.svg",
      artistImage: event.metadata.artistPhoto
    };
  }
}
```

**Total Handler Code Per Partnership**: 30-100 lines
**Time to Add New Partnership**: 1-2 days (not weeks/months)
